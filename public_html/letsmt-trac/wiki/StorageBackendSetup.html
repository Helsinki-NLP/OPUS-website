Compressed Filesystems

Using the filesystem backend for storing the data uses uncompressed files and can easily fill your hard disk when storing large amounts or data collections. Using a compressed file system can reduce the amount of disk space substantially. A simple tool is fusecompress:

  • install the software 
sudo apt-get install libfuse-dev
svn checkout http://fusecompress.googlecode.com/svn/trunk/ fusecompress
cd fusecompress
./autogen.sh
./configure
sudo make install
  • mount a compressed directory 
fusecompress /path/to/dir

Unmounting (and decompressing) the data is done with 

fusermount -u /path/to/dir

More info: ​http://code.google.com/p/fusecompress/ and ​http://code.google.com/p/fusecompress/source/browse/branches/fusecompress-0.9.1.1/README

More options: ​http://code.google.com/p/fusecompress/wiki/Shortcomings

Consider this discussion on performance and robustness: ​https://bugs.launchpad.net/ubuntu/+source/fusecompress/+bug/582943

Add encryption: ​http://www.fell.it/2012/02/24/compressed-and-encrypted-dropbox/

Mounting via fstab: 

fusecompress#/storage  /mount  fuse  user  0  0
