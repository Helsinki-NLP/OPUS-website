Access

  • Requesting the access settings of all branches in a slot, e.g. /access/slot_name_1 
$LETSMT_CONNECT -X GET "$LETSMT_URL/access/slot_name_1?uid=user_id_2"

<letsmt-ws version="3">
  <list path="slot_name_1">
    <entry kind="branch" path="slot_name_1/user_id_2">
      <group>group_id_2</group>
      <owner>user_id_3</owner>
    </entry>
    <entry kind="branch" path="slot_name_1/user_id_3">
      <group>group_id_2</group>
      <owner>user_id_2</owner>
    </entry>
  </list>
  <status code="0" location="/access/slot_name_1" operation="GET" type="ok"></status>
</letsmt-ws>
  • Requesting the access settings of a certain branch, e.g. /access/slot_name_1/user_id_2 
$LETSMT_CONNECT -X GET "$LETSMT_URL/access/slot_name_1/user_id_2?uid=user_id_2"

<letsmt-ws version="3">
  <list path="slot_name_1/user_id_2">
    <entry kind="branch" path="slot_name_1/user_id_2">
      <group>group_id_2</group>
      <owner>user_id_2</owner>
    </entry>
  </list>
  <status code="0" location="/access/slot_name_1/user_id_2" operation="GET" type="ok"></status>
</letsmt-ws>
  • Setting the group name of a branch to a new value, e.g. /access/slot_name_2/user_id_2?uid=user_id_2&gid=group_3 
$LETSMT_CONNECT -X PUT "$LETSMT_URL/access/slot_name_2/user_id_2?uid=user_id_2&gid=group_3"

<letsmt-ws version="3">
  <status code="0" location="/access/slot_name_2/user_id_2" operation="PUT" type="ok">Set group to 'group_3'</status>
</letsmt-ws>

Admin

  • Return the full SVN path of a given resource, e.g. /admin/slot_name_2/user_id_2/test_file_2.xml?type=svnpath 
$LETSMT_CONNECT -X GET "$LETSMT_URL/admin/slot_name_2/user_id_2/test_file_2.xml?uid=user_id_2&type=svnpath"

<letsmt-ws version="3">
  <list path="file:///slot_name_2/user_id_2/test_file_2.xml">
    <entry kind="svn path">file:///var/lib/letsmt/www-data/slot_name_2/user_id_2/test_file_2.xml</entry>
  </list>
  <status code="0" location="/admin/slot_name_2/user_id_2/test_file_2.xml" operation="GET" type="ok"></status>
</letsmt-ws>

Group

  • List all groups: /group 
$LETSMT_CONNECT -X GET "$LETSMT_URL/group?uid=user_id_1"

<letsmt-ws version="3">
  <list path="/">
    <entry id="group_id_1" kind="group">
      <user>user_id_1</user>
    </entry>
    <entry id="group_id_2" kind="group">
      <user>user_id_2</user>
    </entry>
  </list>
  <status code="0" location="/group" operation="GET" type="ok"></status>
</letsmt-ws>
  • Get the user(s) of a group, e.g. /group/group_id_1
    • Question: What should list path show? 
$LETSMT_CONNECT -X GET "$LETSMT_URL/group/group_id_1?uid=user_id"

<letsmt-ws version="3">
  <list path="/group_id_1">
    <entry id="group_id_1" kind="group">
      <user>user_id_1</user>
      <user>user_id_2</user>
    </entry>
  </list>
  <status code="0" location="/group/group_id_1" operation="GET" type="ok"></status>
</letsmt-ws>
  • Add user to a group, e.g. /group/group_id_1/user_2?uid=user_id_1 
$LETSMT_CONNECT -X PUT "$LETSMT_URL/group/group_id_1/user_2?uid=user_id_1"

<letsmt-ws version="3">
  <status code="0" location="/group/group_id_1/user_2" operation="PUT" type="ok">Added user 'user_2' to group 'group_id_1'</status>
</letsmt-ws>
  • Create a group with the effective user as first member, e.g. /group/group_1?uid=user_1 
$LETSMT_CONNECT -X POST "$LETSMT_URL/group/group_1?uid=user_1"

<letsmt-ws version="3">
  <status code="0" location="/group/group_1" operation="POST" type="ok">Created group 'group_1' with user 'user_1'</status>
</letsmt-ws>
  • Delete a group, e.g. /group/group_1?uid=user_1 
$LETSMT_CONNECT -X DELETE "$LETSMT_URL/group/group_1?uid=user_1"

<letsmt-ws version="3">
  <status code="0" location="/group/group_1" operation="DELETE" type="ok"></status>
</letsmt-ws>
  • Show user specific import and alignment parameters 
$LETSMT_CONNECT -X GET "$LETSMT_URL/group/user_id_1?uid=user_id_1&action=showinfo" 

<letsmt-ws version="">
  <list path="group/user_id_1">
    <entry id="user_id_1" kind="user info">
      <AlignPara_method>GaleChurch</AlignPara_method>
      <ImportPara_pdf_mode>raw</ImportPara_pdf_mode>
      <member>user_id_1</member>
      <member_of>public,user_id_1</member_of>
      <my_group>user_id_1</my_group>
    </entry>
  </list>
  <status code="0" location="/group/user_id_1" operation="GET" type="ok"></status>
</letsmt-ws>
  • Set user specific import or alignment parameters (overwrites values of existing keys) 
$LETSMT_CONNECT -X POST "$LETSMT_URL/group/user_id_1?uid=user_id_1&ImportPara_normalizer=moses" 

<letsmt-ws version="">
  <status code="0" location="/group/user_id_1" operation="POST" type="ok">Set user info for user 'user_id_1'</status>
</letsmt-ws>
  • Add user specific import or alignment parameters (adding values to existing keys)

<letsmt-ws version="">

</letsmt-ws>
  • Delete user specific import or alignment parameters (removing key value pairs)

MetaData

  • Searching for meta data by path, e.g. /slot_name_1 
$LETSMT_CONNECT -X GET "$LETSMT_URL/metadata/slot_name_1?uid=user_id_1"

<letsmt-ws version="">
  <list path="">
    <entry path="slot_name_1">
      <creator>user_id_1</creator>
      <resource-type>slot</resource-type>
    </entry>
  </list>
  <status code="0" location="/metadata/slot_name_1" operation="GET" type="ok">Found matching path ID. Listing all of its properties</status>
</letsmt-ws>
  • Searching for meta data by argument, without any path, only IDs as result, e.g. status=imported
    • Questions: should the path attribute in the list tag show anything here? 
$LETSMT_CONNECT -X GET "$LETSMT_URL/metadata?uid=user_id_2&status=imported"

<letsmt-ws version="3">
  <list path="">
    <entry path="slot_name_2/user_id_2/uploads/test_file_1.txt" />
    <entry path="slot_name_2/user_id_2/uploads/test_file_2.txt" />
  </list>
  <status code="0" location="/metadata" operation="GET" type="ok">Found 2 matching entries</status>
</letsmt-ws>
  • Searching for meta data by path and argument, only IDs as result, e.g. /slot_name_2/test_file_2.xml?uid=user_id_2&size
    • Question: Only the by argument requested properties are returned but also the 'uid' property. Should 'uid' be always included? 
$LETSMT_CONNECT -X GET "$LETSMT_URL/metadata/slot_name_2/test_file_2.xml?uid=user_id_2&size"

<letsmt-ws version="3">
  <list path="">
    <entry path="slot_name_2/test_file_2.xml" />
  </list>
  <status code="0" location="/metadata/slot_name_2/test_file_2.xml" operation="GET" type="ok">Found matching path ID. Listing only the requested properties</status>
</letsmt-ws>
  • Searching for meta data by argument, without any path, full result, e.g. status=imported&action=list_all
    • Questions: should the path attribute in the list tag show anything here? 
$LETSMT_CONNECT -X GET "$LETSMT_URL/metadata?uid=user_id_2&status=imported&action=list_all"

<letsmt-ws version="3">
  <list path="">
    <entry path="slot_name_2/user_id_2/uploads/test_file_1.txt">
      <status>imported</status>
      <uid>user_id_2</uid>
    </entry>
    <entry path="slot_name_2/user_id_2/uploads/test_file_2.txt">
      <status>imported</status>
      <uid>user_id_2</uid>
    </entry>
  </list>
  <status code="0" location="/metadata" operation="GET" type="ok">Found 2 matching entries</status>
</letsmt-ws>
  • Searching for meta data by path and argument, full result, e.g. /slot_name_2/test_file_2.xml?uid=user_id_2&size&action=list_all
    • Question: Only the by argument requested properties are returned but also the 'uid' property. Should 'uid' be always included? 
$LETSMT_CONNECT -X GET "$LETSMT_URL/metadata/slot_name_2/test_file_2.xml?uid=user_id_2&size&action=list_all"

<letsmt-ws version="3">
  <list path="">
    <entry path="slot_name_2/test_file_2.xml">
      <size>5</size>
      <uid>user_id_2</uid>
    </entry>
  </list>
  <status code="0" location="/metadata/slot_name_2/test_file_2.xml" operation="GET" type="ok">Found matching path ID. Listing only the requested properties</status>
</letsmt-ws>

  • Adding new or extending existing meta data entry, e.g. /slot_name_2/test_file_2.xml?uid=user_id_2&foo=bar

Storage

  • List all slots, /storage?uid=bob 
$LETSMT_CONNECT -X GET "$LETSMT_URL/storage?uid=bob"

<letsmt-ws version="3">
  <list path="file:///">
    <entry kind="slot">
      <name>slot_name_1</name>
      <group>users</group>
      <owner>root</owner>
      <perm>drwrwrw</perm>
    </entry>
    <entry kind="slot">
      <name>slot_name_2</name>
      <group>users</group>
      <owner>root</owner>
      <perm>drwrwrw</perm>
    </entry>
</list>
  <status code="0" location="/storage" operation="GET" type="ok"></status>
</letsmt-ws>
  • List all branches of a slot, e.g. /storage/slot_name_2?uid=user_id_2 
$LETSMT_CONNECT -X GET "$LETSMT_URL/storage/slot_name_2?uid=user_id_2"

<letsmt-ws version="3">
  <list path="file:///slot_name_2">
    <entry kind="branch">
      <name>user_id_1</name>
      <group>group_id_2</group>
      <owner>user_id_1</owner>
      <perm>rwr---</perm>
    </entry>
    <entry kind="branch">
      <name>user_id_2</name>
      <group>group_id_2</group>
      <owner>user_id_2</owner>
      <perm>rwr---</perm>
    </entry>
  </list>
  <status code="0" location="/storage/slot_name_2" operation="GET" type="ok"></status>
</letsmt-ws>

$LETSMT_CONNECT -X GET "$LETSMT_URL/storage/slot_name_2/user_id_2/uploads/test_file_2.txt?uid=user_id_2&action=cat&from=1&to=3"

<letsmt-ws version="3">
  <list path="">
    <entry>This is just a test file
Nothing more
Nothing less
</entry>
  </list>
  <status code="0" location="/storage/slot_name_2/user_id_2/uploads/test_file_2.txt" operation="GET" type="ok"></status>

<letsmt-ws version="3">
  <status code="0" location="/storage/slot_name_2/user_id_2/uploads/test.txt" operation="PUT" type="ok">add ok /slot_name_2/user_id_2/uploads/test.txt</status>

  • Upload files with import, e.g. /storage/slot_name_2/user_id_2/uploads/test.txt?uid=user_id_2&action=import

  <status code="0" location="/storage/slot_name_2/user_id_2/uploads/test.txt" operation="PUT" type="ok">add ok /slot_name_2/user_id_2/uploads/test.txt</status>
</letsmt-ws>
  • Delete a resource

  <status code="0" location="/storage/slot_name_2" operation="DELETE" type="ok">Deleted /storage/slot_name_2</status>
