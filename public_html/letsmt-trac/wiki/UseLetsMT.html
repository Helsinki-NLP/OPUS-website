Using The LetsMT Resource Repository Software

The easiest way to use the software is by using letsmt scripts:

  • ​letsmt_rest
  • ​letsmt_import
  • ​letsmt_fetch
  • ​letsmt_convert
  • ​letsmt_tokenize
  • ​letsmt_detokenize
  • ​letsmt_align
  • ​letsmt_chardetect
  • ​letsmt_langdetect

letsmt_rest

This is a frontend to most calls in the webservice API. For more information about all its options, please, look at the ​manpages:

man letsmt_rest

Here are some common usage examples:

  • create a new user
  • create a new slot
  • upload (and import) a TMX file
  • check data uploads, imports and metadata 
letsmt_rest -u username create_user
letsmt_rest -u username -s slotname create
letsmt_rest -u username -s slotname import tmx/MyMemory.tmx

# check whether the file has been uploaded:
letsmt_rest -u username -s slotname -p uploads/tmx show

# check its metadata (check import progress/problems etc):
letsmt_rest -u username -s slotname -p uploads/tmx/MyMemory.tmx showmeta

# wait for the import to finish ... and check if appropriate XML files are created:
# (xx and yy are the language IDs used in your TMX; xx and yy should be alphabetically sorted)
letsmt_rest -u username -s slotname -p xml show
letsmt_rest -u username -s slotname -p xml/xx show
letsmt_rest -u username -s slotname -p xml/yy show
letsmt_rest -u username -s slotname -p xml/xx-yy show

The latest version of letsmt_rest can also handle revsions using '-r' or using '@xx'-extensions to the resource path (only if a version-controlled backend is used). For example, you can fetch revision 4 of the resource xml/en using the following commands: 

# fetch revision 4 of xml/en:
letsmt_rest -u username -s slotname -p xml/en -r 4 fetch

# the same using the path-extension syntax
letsmt_rest -u username -s slotname -p xml/en@4 fetch

The same works with letsmt_rest commands like 'show' and 'cat'. The -r flag can also be used to show the entire history of a given resource, for example ('HEAD' refers to the last revision of the resource): 

# list information including revision history for xml/en in slot slotname:
letsmt_rest -u username -s slotname -r HEAD -d xml/en show

letsmt_import

This is the script that will do the automatic document imports. It will be called internally but you can also run it manually on the command line. For more details on its usage check the ​manpages:

man letsmt_import

Here is an example:

  • assume that you have a pdf file 'mypdf.pdf' in your slot 'slotname' that needs to be imported
  • Important: all uploads should be in the uploads directory
  • Important: sub-directories in uploads should correspond to the file format (pdf in our case)
  • Important: monolingual files should be in sub-directories corresponding to the language of their contents (for example 'en' for English) 
# upload the pdf file and check if the it is really there:
letsmt_rest -u username -s slotname -p uploads/pdf/en upload mypdf.pdf
letsmt_rest -u username -s slotname -p uploads/pdf/en show

# import with standard settings
letsmt_import -u username -s slotname -p uploads/pdf/en/mypdf.pdf

# check if the import was successful
letsmt_rest -u username -s slotname -p uploads/pdf/en/mypdf.pdf showmeta

# check if the XML file was created
letsmt_rest -u username -s slotname -p xml/en/mypdf.xml show
letsmt_rest -u username -s slotname -p xml/en/mypdf.xml showmeta

# show the first 10 lines of the XML code
letsmt_rest -u username -s slotname -p xml/en/mypdf.xml -T 10 cat

Some import handlers allow alternative modes. Try, for example, the 'layout' mode for the PDF converter: 

# import with standard settings
letsmt_import -u username -s slotname -p uploads/pdf/en/mypdf.pdf -m layout

# show the first 10 lines of the XML code
letsmt_rest -u username -s slotname -p xml/en/mypdf.xml -T 10 cat

letsmt_langdetect

This is a tool for language detection. Here are some common usage examples: 

# detect the language in a given text file
letsmt_langdetect myfile.txt

# detect the language of a repository resource
letsmt_langdetect -u username -s slotname -p relative/path/to/myfile.txt

# detect the language of a resource in a different format (will be converted to text first!)
letsmt_langdetect -u username -s slotname -p uploads/pdf/en/mypdf.pdf

# create a new language model for language 'lang' from text file 'myfile.txt'
letsmt_langdetect -n lang myfile.txt

The last example tries to install the language model in the central LM home directory. This fails if you do not have permissions to create files there or if the file already exists. In that case, letsmt_langdetect will try to safe the file in the current directory. 'lang' can be a valid ISO639 code or a valid language name. Internally, this will be converted into a ISO639 2-letter code if possible. This code will be used to store the language model (for example 'en.lm' for English).

Look also at the ​manpages:

man letsmt_langdetect
