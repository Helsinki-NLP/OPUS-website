Extended functionality & devlopment ideas

New storage backends

  • other version control systems (git, hg ...)
  • combined svn and filesystem
    • keep a working copy and always update the working copy only
    • implement commit function in storage API (user can decide when to commit the data from the working copy)
    • when uploading files: mark in metadata that the version is not committed yet
    • commit for the entire working copy (commit all changes in one step; use background job for that)
    • implement status function in storage API (/storage/slot/branch/status) to show status of the working copy (svn status)
  • compressed filesystem (use fusecompress)

User/Domain-specific tools

  • train sentence splitters
  • train tokenizer
  • use user-specific pre-processing tools in data import

Interactive sentence alignment

  • Integrate tools for improving sentence alignment
  • use something like ICA

User feedback

  • rating sentence alignment quality
  • rating corpus quality
  • tag corpora and other resources

Upload of language models

  • allow to upload language models in a standard ARPA format
  • advantage: don't need to create language models at our server
  • no IPR problems with data that have been used for creating those models

Automatic domain classification

  • trust automatic classification more than user input?
  • problem: language specific classifiers!
  • interesting idea: find similar documents and classify in the same way (some kind of kNN?)

Questions: Who? How? When?

Automatic data selection

  • related to the automatic classification ideas
  • use IR techniques to find documents similar to given translation examples
  • user interface: "I want to translate sentence like those - find me appropriate parallel training data with at least 1 million tokens!"
  • (always interpolate domain-specific TMs and LMs with large general-purpose backoff models!!!)

Integration with TDA (TAUS data)

  • Integrate TDA for training SMT models
  • API to access TDA data
  • no storage of TDA data in our repository (only temporary for training purposes!)

Questions: Who? How? TDA API?
