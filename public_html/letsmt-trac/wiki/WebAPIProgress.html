Ideas for extensions of the higher level storage

  • integration of metadata handling and branch access permissions (groups) via high-level requests to /letsmt/
Method Path Arguments Upload Download Description
POST /letsmt/slot [uid:user]([gid:group][key:value])* Content Status create slot/user, set group permissions to <group> and add <key>-<value> pairs to metadatabase for slot/user
POST /letsmt/slot/path [uid:user]([key:value])* Content Status create slot/user/path, and add <key>-<value> pairs to metadatabase for slot/user, ignore 'gid'!!
POST/PUT/GET/DELETE /letsmt/path [type:meta]([key:value])* Status pass through to /metadata/path (with all arguments except type!)
PUT /letsmt/slot [uid:user][type:access][gid:group] Status pass through to /groups/slot (with all arguments except type!)

Open issues and other comments

Jag checkar snart in en ny version av repositoriet, med mer saker som fungerar (se sådant jag ändrar till "works" ovan fram tills jag checkar in.

  • LETSMTVERSION ändras från 1 till 2
  • conf.sh kommer inte längre vara nödvändig för Webbervicen (men fortfarande för kommandoradsverktyg). Webbservicen har sina miljövariabler satta i /etc/apache2/sites-available/letsmt istället.
  • mod_perl aktiverat, för att snabba upp webbservicen lite. Under en tid kommer det betyda att vissa saker i loggfilen (som i letsmtversion 1 hette /tmp/pw_test.log) inte kommer inkludera sådant som webbservicen skrev till STDERR. Det går att åtgärda, men är inte högprioriterat.
  • "mountpoint" för "diskarna" flyttad till /opt/letsmt_diskroot (LETSMTDISKROOT)
