CONFIG

conf.sh

The central config file.

  • DBHOST - MySQL host
  • DBNAME - MySQL database name
  • DBPASS - MySQL password
  • DBPORT - MySQL port
  • DBUSER - MySQL user
  • LETSMTCONF - the configuration file in use
  • LETSMTDISKROOT - the top directory under which Subversion creates slot repositories
  • LETSMTHOST - the hostname of the repository
  • LETSMTLOG_DIR - the log file directory, configuration of logging is in /log4perl.conf
  • LETSMTPORT - the port of the repository
  • LETSMTPUBLICGROUP - the name of the group used for public resources
  • LETSMTROOT - the root/prefix of the installed software
  • LETSMTUSER - the user running the repository. For Ubuntu installations, this is the same as the user running Apache2, but can also be configured for development users
  • LETSMTVERSION - the version of the data structures and database definitions. If these are changed, this version number should also be changed
  • LETSMTVIRTHOSTFILE - the name to use for the virtual hosts file
  • LETSMT_CACERT - the CA certificate (commonly named ca.crt.
  • LETSMT_CERTPATH - corresponds to Apache2 setting SSLCACertificatePath
  • LETSMT_CERTROOT - the directory holding LETSMT_CACERT (ca.crt)
  • LETSMT_CONNECT - a curl-call template, using the function curlfilter from admin/functions.sh
  • LETSMT_CONNECT_RAW - a curl-call
  • LETSMT_MODPERL_STARTUP - the mod_perl startup file, default name conf_modperl.pl
  • LETSMT_REPOS_DIR_UPLOAD - the reserved name for the upload directory, when uploading to the higher level (/letsmt)
  • LETSMT_REVOCATIONPATH - corresponds to Apache2 setting SSLCARevocationPath
  • LETSMT_SSL_CITY - the city (ST), used in certificate signature requests
  • LETSMT_SSL_COMPANY - the company (O), used in certificate signature requests
  • LETSMT_SSL_COUNTRY - the country (OU), used in certificate signature requests
  • LETSMT_SSL_DIR - deprecated
  • LETSMT_SSL_SUBDIR - directory holding the repository's SSL files
  • LETSMT_SSL_USER - username of the connecting client, for which a certificate (LETSMT_USERCERT) will be generated
  • LETSMT_SSL_USERNAME - the email address of LETSMT_SSL_USER
  • LETSMT_TAR_INITREPOS - the initial contents of a new slot is stored in this tar-file
  • LETSMT_URL - the formatted URL of the web service
  • LETSMT_USERCERT - the connecting client's user certificate
  • LETSMT_USERCERTPASS - the password for the LETSMT_USERCERT certificate
  • LETSMT_USERKEY - the key corresponding to LETSMT_USERCERT
  • LOG4PERLCONF - the log configuration, see ​http://log4perl.sourceforge.net/ for documentation
  • UPLOADDIR - the temporary directory holding files being uploaded
  • VC_BACKEND - the backend storage module to use. Currently only VCSubversion is implemented.

conf_modperl.pl

This is read at apache initiation of mod_perl (apache reload and apache restart).

admin/qconf_add_queue_editor.sh

This is used by LetsMT::Repository::QueueManager to create a new queue.

BUILD-FILES

Makefile

Main Makefile used for installs, updates, upgrades and testing.

installation/Makefile

Backend Makefile for the gritty details. Called from the main Makefile.

installation/Makefile.prereqs

Used for installing prerequisites on a new server.

INSTALLATION

admin/setup_cert_based_auth.pl

This is used by installation/Makefile to create ssl certificates.

installation/MyConfig.pm

This is used by installation/Makefile.prereqs to setup CPAN.

installation/setup_db.sh

This is used by installation/Makefile to setup MySQL.

installation/letsmt

This is an Apache2 virtual host file template, instantiated by installation/Makefile when make install.

WEB SERVICE

perllib/LetsMT/lib/LetsMT/WWW/LetsMT_modperl_handler.pm

This is the Apache2 mod_perl handler.

perllib/LetsMT/lib/LetsMT/WWW/LetsMT_modperl_handler_exception.pm

This implements a mod_perl-friendly exception used in the resource repository code.

REPOSITORY

perllib/LetsMT/lib/LetsMT/Repository/Err.pm

The central place where error types are defined, and the raise-subroutine is defined.

perllib/LetsMT/lib/LetsMT/Repository/Persist.pm

The super class for modules with object persistence.

perllib/LetsMT/lib/LetsMT/Repository/QueueManager.pm

The interface to Sun Grid Engine.

perllib/LetsMT/lib/LetsMT/Repository/Safesys.pm

Some helper routines for interfacing with the operating system.

perllib/LetsMT/lib/LetsMT/Repository/VCSubversion.pm

The interface to Subversion.

perllib/LetsMT/lib/LetsMT/Repository/AdminManager.pm

Nearly unused. Implements a total reset subroutine (obviously, use with care).

perllib/LetsMT/lib/LetsMT/Repository/GroupManager.pm

Manager for working with groups.

perllib/LetsMT/lib/LetsMT/Repository/GroupManager/Group.pm

Persistent part of GroupManager?. Implements groups.

perllib/LetsMT/lib/LetsMT/Repository/GroupManager/Groupmember.pm

Persistent part of GroupManager?. Implements user group memberships.

perllib/LetsMT/lib/LetsMT/Repository/StorageManager.pm

Manager for working with storage.

perllib/LetsMT/lib/LetsMT/Repository/StorageManager/Branch.pm

Persistent part of StorageManager?. Implements branches.

perllib/LetsMT/lib/LetsMT/Repository/StorageManager/Partition.pm

Persistent part of StorageManager?. Implements partitions. This is a candidate for refactoring and removal, as it basically does nothing of value.

perllib/LetsMT/lib/LetsMT/Repository/StorageManager/Slot.pm

Persistent part of StorageManager?. Implements slots.

perllib/LetsMT/lib/LetsMT/WWW/File.pm

Subroutines for safe file handling.

TESTING

The goal of testing framework is to carry a complete set of tests that can be used to verify the functionality of the resource repository.

admin/execute_in_order.sh

Helper script to execute the different tests in the right order, and optionally selecting which tests to run.

admin/functions.sh

Defines a set of useful bash functions. The testing framework depend on a number of these.

tests/10_groups.sh

Testing groups.

tests/15_usr.sh

Testing users.

tests/20_directories.sh

Testing directories.

tests/21_branching.sh

Testing branching.

tests/30_single_files.sh

Testing single file handling.

tests/31_single_files_letsmtrepos.sh

Testing higherlevel API file handling.

tests/32_filetree.sh

Testing with many files.

tests/34_metadata.sh

Testing metadata.
