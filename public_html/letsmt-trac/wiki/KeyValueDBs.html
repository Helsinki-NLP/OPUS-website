A list of links to Key / Value DBs and related tools & info

Very promising: Tokyo Cabinet!

  • ​http://www.igvita.com/2009/02/13/tokyo-cabinet-beyond-key-value-store/
  • ​http://fallabs.com/
  • ​http://fallabs.com/tokyocabinet/
  • ​http://www.engineyard.com/blog/2009/key-value-stores-in-ruby/
  • use a table database (TokyoCabinet::TDB) with arbitrary columns! Cool!
  • even supports keyword search within string fields with boolean operators! example perl script
  • install: sudo apt-get install libbz2-dev libtokyocabinet-dev and Perl API from ​http://fallabs.com/tokyocabinet/perlpkg/tokyocabinet-perl-1.34.tar.gz
  • on MacOSX: port install tokyocabinet but problems with Perl API (​http://efreedom.com/Question/1-1537937/Tokyo-Cabinet-Perl-Api-Libtokyocabinetdylib-File-Required-Architecture) probably need to modify cc call (add -arch x86_64)
  • PHP interfaces:
    • ​http://mamasam.indefero.net/p/tyrant/
    • ​http://code.google.com/p/phptyrant/

Maybe we should look at Kyoto Cabinet as well:

  • ​http://fallabs.com/kyotocabinet/
  • ​http://fallabs.com/mikio/tech-en/promenade.cgi?id=7
  • (does it support table DBs like Tokyo Cabinet?)

Another way: use full-text search in mySQL databases!

  • key-value pairs = tokens in a full-text field
  • create tables with ​TEXT fields and index if type FULLTEXT!
  • create boolean queries:
  • ​http://dev.mysql.com/doc/refman/5.0/en/fulltext-boolean.html
  • problem: spaces in keys & values, handling of stopwords & frequent words in mySQL, numerical values (and numerical comparisons) ...

  • simple Perl API: ​http://search.cpan.org/~norbu/Store-CouchDB-1.5/lib/Store/CouchDB.pm
  • ​http://search.cpan.org/~beppu/AnyEvent-CouchDB-1.22/lib/AnyEvent/CouchDB/Database.pm
  • not Couch but similar: ​http://search.cpan.org/~fujiwara/DBIx-CouchLike-0.16/lib/DBIx/CouchLike.pm

other:

Inverted Index files and IR technology (store keys-value pairs as tokens in a document!)

  • looks simple & easy: ​http://search.cpan.org/~lbrocard/Search-Mousse-0.32/lib/Search/Mousse.pm
  • simple indexer (qdbm-based): ​http://search.cpan.org/~ikebe/Search-Odeum-0.02/lib/Search/Odeum.pm
  • QDBM includes inverted index handling: ​http://search.cpan.org/~yamato/QDBM_File-1.12/lib/QDBM_File.pm (doesn't compile on amd64)
  • for HTML-docs: ​http://search.cpan.org/~awrigley/HTML-Index-0.15/lib/HTML/Index.pm
  • a bit strange: ​http://search.cpan.org/~snowhare/Search-InvertedIndex-1.14/lib/Search/InvertedIndex.pm
