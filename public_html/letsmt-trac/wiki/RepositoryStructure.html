The Basic Structure of the LetsMT data repository

  • each corpus has a unique name (CORPUSNAME)
  • each corpus has an upload directory (where all user-contributed files will be stored in their original format)
  • each corpus has an xml directory for storing all documents in the internal LetsMT format
    • there are subdirectories for each language (using ISO-language IDs)
    • there are subdirectories for each language pair for storing sentence alignment between parallel documents
    • files with the same name in different language subdirectories are parallel!

Internally the a corpus will be stored in a branch using the user-id as its name. This supports data sharing with data integrity as explained here. 

storage/CORPUSNAME_1
storage/CORPUSNAME_1/USER_1
storage/CORPUSNAME_2
storage/CORPUSNAME_2/USER_2
storage/CORPUSNAME_2/USER_3
...

Within a corpus the file structure looks like this (a simple example - omitting the branch name): 

CORPUSNAME/
CORPUSNAME/uploads
CORPUSNAME/uploads/tmx
CORPUSNAME/uploads/tmx/first.tmx
CORPUSNAME/uploads/tmx/second.tmx
...
CORPUSNAME/uploads/xliff
...
CORPUSNAME/uploads/doc
CORPUSNAME/uploads/doc/en/word.doc
CORPUSNAME/uploads/doc/fr/word.doc
CORPUSNAME/uploads/doc/de/word.doc
...
CORPUSNAME/uploads/pdf
...
CORPUSNAME/xml/de
CORPUSNAME/xml/de/word.xml
CORPUSNAME/xml/en
CORPUSNAME/xml/en/first.xml
CORPUSNAME/xml/en/second.xml
CORPUSNAME/xml/en/word.xml
CORPUSNAME/xml/fr
CORPUSNAME/xml/fr/first.xml
CORPUSNAME/xml/fr/second.xml
CORPUSNAME/xml/fr/word.xml
...
CORPUSNAME/xml/de-fr.xml
CORPUSNAME/xml/de-fr/word.xml
CORPUSNAME/xml/de-en.xml
CORPUSNAME/xml/de-en/word.xml
CORPUSNAME/xml/en-fr.xml
CORPUSNAME/xml/en-fr/first.xml
CORPUSNAME/xml/en-fr/second.xml
CORPUSNAME/xml/en-fr/word.xml
...

The top-level sentence-alignment file contains sentence alignments of the entire corpus! For example, "CORPUSNAME/xml/fr-en.xml" includes all sentence alignments between the "first", "second" and "word" documents in French and English.

Sentence alignment is symmetric! Therefore, we store only alignments in one direction according to the alphabetic order of the language IDs! More information about the data formats can be found here.

Ideas about the model repository

General idea: Use the same implementation for storing SMT models and other resources

  • Another layer with the same functionality?! 
<host>/storage/    for corpora
<host>/models/    for SMT models

It could be even good to have different places for TM, LM and other SMT model files to make it flexible to re-use LMs and TMs in combinations with others (and also for the coming incremental training procedures) 

<host>/models/tm            translation models (phrase tables,
reordering models)
<host>/models/lm            language models
<host>/models/moses      moses configutration files and parameters, ....

And inside of each resource we sort by language/language-pair: 

<host>/models/tm/<srclang>/<trglang>/<resourcename>
<host>/models/lm/<lang>/<resourcename>
<host>/models/moses/<srclang>/<trglang>/<resourcename>

This is without branching for the time being. But branching could be useful here again. This could be used here as well to make model sharing possible!
