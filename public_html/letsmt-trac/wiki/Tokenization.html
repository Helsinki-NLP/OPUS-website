Tools for tokenization

Generic tools / multiple languages

  • New trainable tokenizer:
    • ​https://github.com/jirkamarsik/trainable-tokenizer
  • Moses/Europarl? tokenizer
    • ​http://www.statmt.org/wmt10/scripts.tgz
    • more ​non-breaking prefixes
  • OpenNLP (GPL?)
    • ​http://opennlp.sourceforge.net/
    • trainable tokenizer,
    • models available for English, German, Spanish, Thai
  • NLTK (Apache license)
    • (regexp)
    • ​http://www.nltk.org/
  • huntoken (License?)
    • ​http://mokk.bme.hu/resources/huntoken
  • Jena NLP tools
    • ​http://www.julielab.de/Resources/Software/NLP+Tools.html
    • trainable
    • article: ​http://books.google.com/books?id=dEXxG4XyvKwC&pg=PA524&lpg=PA524&dq=Katrin+Tomanek,+Joachim+Wermter,+and+Udo+Hahn.+A+reappraisal+of+sentence+and+token+splitting+for+life+science+documents.+In+MEDINFO+2007+-+Proceedings+of+the+12th+World+Congress+on+Medical+Informatics.&source=bl&ots=AvAJreMFHq&sig=ASKHCYHUHiXC7IBbhErmF1MGTJw&hl=en&ei=Qu0MTJi-Cd-JOLndtRQ&sa=X&oi=book_result&ct=result&resnum=1&ved=0CBIQ6AEwAA#v=onepage&q&f=false
  • FreeLing? (GPL)
    • ​http://www.lsi.upc.edu/~nlp/freeling
    • regexp tokenizer
    • (mainly Catalan & Spanish?)
  • another classifier-based tokenizer;
    • ​http://labs.sapo.pt/up/2011/11/12/sylvester-ugc-tokenizer/
    • ​http://dl.acm.org/citation.cfm?id=1871853
  • Ellogon (LGPL)
    • ​http://www.ellogon.org
    • (useful but seems to big/complex for our purposes(?))
  • Java 6, BreakIterator?
    • ​http://java.sun.com/javase/6/docs/api/java/text/BreakIterator.html#word
    • comment: "Actually I have only used the BreakIterator? for getting word and sentence instances from Portuguese text and it works quite well. For instance, I don't remember of having any problem with the word instance tokenization. As for the sentence splitting there might be a few problems, but it still works ok. For instance, I remeber that it could recognize dots in references, such as Author et al. 2010, and it did not split the sentence with the dot there. Furthermore, when using the getCharacterInstance(), getWordInstance() and getSentenceInstance() methods you can provide the desired Locale (there are several available countries), and I believe that splitting will adapt to it."
