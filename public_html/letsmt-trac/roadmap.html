Roadmap

Milestone: MS2 - Initial training data

10 years late (Feb 27, 2011, 11:00:00 PM)

Initial training data

  • leading partner: UCPH
  • WP4

Initial training data (D4.3, D4.5 and D4.7) collected

  • 6 language pairs
  • 9 million sentence pairs in total

Question: Who is asking EC for IPR for EU material (Europarl, EMEA)

Extended goals: robust TMX upload in repository prototype:

  • robust upload of TMX data
  • creation of validation & conversion jobs at backend
  • robust pre-processing at backend + checkin and database update

Milestone: MS3.1 - New internal API structure

10 years late (Apr 7, 2011, 10:00:00 PM)

  • new API modules
  • new mod_perl handler
  • same functionality as before (but clean implementation)

Milestone: MS3.2 - Integrated sentence alignment

10 years late (Apr 28, 2011, 10:00:00 PM)

  • sentence alignment via off-line processes
  • job management via API
  • integration of sentence alignment as webservice in the repository

Milestone: MS3 - Resource repository

9 years late (Aug 30, 2011, 10:00:00 PM)

Resource repository populated with initial training data

  • leading partner: UUP
  • WP2

Training data for 10 language pairs and 3 domains are collected and processed, D2.3

Sub-goal 1: Robust REST API

  • completely implemented REST API for LetsMT
  • documentation of REST requests
  • documentation of setup & administration
  • running REST server for experiments

Sub-goal 2: Demo-interface to repository prototype

  • access to LetsMT repository
  • simple web interface for several tasks
    • login
    • create resource
    • upload resources (TMX)
    • browsing resources and searching for specific languages & language pairs / domains, ...
  • selecting resources for SMT training

Milestone: MS4 - Initial SMT models

9 years late (Aug 30, 2011, 10:00:00 PM)

Initial training of SMT systems

  • leading partner: UEDIN
  • WP3

Extended goals:

  • running REST server with initial datasets (from OPUS)
  • user authentication and user management
  • repository with search functions and resource selection
  • data sharing possibilities
  • access to status information
  • running backend for various jobs
