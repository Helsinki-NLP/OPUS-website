NAME

25_Repository_API_Job.t - test Job API

DESCRIPTION

This script tests the following assertions:

Calling the Job API via the WebService module, ...

  • after you upload a job-description file to a slot ... (put_file)

  • you can submit the file as a job, ... (post_job)

  • ... which will have the job executed on the SGE grid engine and finished within a reasonable amount of time.

  • You can query metadata of a job description that has been finished ... (get_meta)

  • ... and it will show successful commands (ls, date, ...) as 'done' ...

  • ... and failed commands (e.g. cd to nonexisting directory) as 'failed'.

  • you can resubmit a job description file. (put_job)

  • you can check the status of a (re)submitted job -- which will be 'pending' during job execution. (get_job)

  • The metadata of a (re)submitted job description will also show the status, as well as the job ID.

  • you can delete a job ... (del_job)

  • ... and it will no longer be found in the system. (get_job)
