Trac encodes all text using ​UTF-8, including text in tickets and wiki pages. Internal processing of text uses true ​Unicode representations. As such, it supports most (all?) commonly used character encodings.

If the default encoding in your source code repository is not UTF-8, you can specify it in the trac.ini: 

default_charset = gbk

Also ensure that your ​database stores its data in UTF-8, otherwise results may be unpredictable.

To convert your database to UTF-8, the easiest way is to create a dump of the database, convert it into UTF-8, for example using ​iconv, and then import it back into the database.
