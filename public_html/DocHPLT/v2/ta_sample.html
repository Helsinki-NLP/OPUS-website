    <P id="2">
        <s id="2.1">மேனிஃபெஸ்ட்டின் இரண்டாவது பதிப்பைச் சேர்ப்பதோடு, பிரபலமான விளம்பரத் தடுப்பான் யுபிளாக் ஆரிஜின் இணைக்கப்பட்டுள்ளது, இது வெப்ரீக்வெஸ்ட் ஏபிஐ -யின் தடுப்பு முறைக்கு ஆதரவு முடிவடைந்ததால், மேனிஃபெஸ்ட்டின் மூன்றாவது பதிப்பிற்கு மாற்ற முடியாது.</s>
    </P>
    <P id="3">
        <s id="3.1">ஜனவரி 17, 2022 நிலவரப்படி, மேனிஃபெஸ்ட்டின் இரண்டாவது பதிப்பைப் பயன்படுத்தும் செருகுநிரல்கள் இனி Chrome இணைய அங்காடியில் ஏற்றுக்கொள்ளப்படாது, ஆனால் முன்பு சேர்க்கப்பட்ட செருகுநிரல் டெவலப்பர்கள் இன்னும் புதுப்பிப்புகளை இடுகையிட முடியும்.</s>
    </P>
    <P id="4">
        <s id="4.1">ஜனவரி 2023 இல், Chrome இரண்டாவது பதிப்போடு இணக்கமாக இருப்பதை நிறுத்திவிடும் மேனிஃபெஸ்ட் மற்றும் அதனுடன் இணைக்கப்பட்ட அனைத்து செருகுநிரல்களும் வேலை செய்வதை நிறுத்தும். அதே நேரத்தில், Chrome இணைய அங்காடியில் இதுபோன்ற துணை நிரல்களுக்கான புதுப்பிப்புகளை இடுகையிடுவது தடைசெய்யப்படும்.</s>
    </P>
    <P id="5">
        <s id="5.1">இந்த ஆண்டின் தொடக்கத்தில், குரோம் 88 க்காக, Chrome நீட்டிப்பு சுற்றுச்சூழல் அமைப்பிற்கான புதிய மேனிஃபெஸ்ட் பதிப்பு கிடைப்பதை அறிவித்தோம். தயாரிப்பில் பல ஆண்டுகளாக, மேனிஃபெஸ்ட் வி 3 அதன் முன்னோடிகளை விட மிகவும் பாதுகாப்பானது, திறமையானது மற்றும் தனியுரிமை பாதுகாக்கும். இது நீட்டிப்பு தளத்தின் பரிணாமம் ஆகும், இது மாறிவரும் வலை நிலப்பரப்பு மற்றும் உலாவி நீட்டிப்புகளின் எதிர்காலம் இரண்டையும் கணக்கில் எடுத்துக்கொள்கிறது.</s>
    </P>
    <P id="6">
        <s id="6.1">நாங்கள் எதிர்காலத்தைப் பார்க்கும்போது, மேனிஃபெஸ்ட் வி 3 இன் செயல்பாட்டை மீண்டும் மீண்டும் செய்ய மற்றும் மேம்படுத்தும்போது, மேனிஃபெஸ்ட் வி 2 இலிருந்து நீட்டிப்புகளை படிப்படியாக நீக்குவதற்கான திட்டத்தின் விவரங்களையும் பகிர்ந்து கொள்ள விரும்புகிறோம்.</s>
    </P>
    <P id="7">
        <s id="7.1">அதை நாம் நினைவில் கொள்ள வேண்டும் மேனிஃபெஸ்ட்டின் மூன்றாவது பதிப்பு, இது திறன்களையும் வளங்களையும் வரையறுக்கிறது செருகுநிரல்களுக்கு வழங்கப்பட வேண்டும், மாறாக பாதுகாப்பு மற்றும் தனியுரிமையை வலுப்படுத்தும் முயற்சியின் ஒரு பகுதியாக webRequest API, வரையறுக்கப்பட்ட திறன்களைக் கொண்ட அறிவிப்பு NetRequest API, முன்மொழியப்பட்டது.</s>
    </P>
    <P id="8">
        <s id="8.1">என்றாலும் WebRequest API உங்கள் சொந்த கட்டுப்படுத்திகளை இணைக்க அனுமதிக்கிறது நெட்வொர்க் கோரிக்கைகளுக்கு முழு அணுகல் மற்றும் பறக்கும்போது போக்குவரத்தை மாற்றக்கூடியவர்கள், அறிவிப்பு NetRequest API ஒரு வடிகட்டுதல் இயந்திரத்திற்கு மட்டுமே அணுகலை வழங்குகிறது உலாவியில் கட்டப்பட்ட பெட்டிக்கு வெளியே தடுக்கும் விதிகளைத் தானே கையாளுகிறது. இது உங்கள் சொந்த வடிகட்டுதல் வழிமுறைகளைப் பயன்படுத்த அனுமதிக்காது மற்றும் நிபந்தனைகளின் அடிப்படையில் ஒன்றோடொன்று ஒன்றிணைந்த சிக்கலான விதிகளை அமைக்க அனுமதிக்காது.</s>
    </P>
    <P id="9">
        <s id="9.1">இந்தத் தேதிகள் நெருங்குகையில், மாற்றத்திற்கு இலக்காகக் கொண்ட Chrome இன் பதிப்பைப் பற்றிய கூடுதல் விவரங்களையும், நீட்டிப்பு டெவலப்பர்கள் மற்றும் பயனர்கள் எவ்வாறு பாதிக்கப்படலாம் என்பதைப் பற்றிய கூடுதல் தகவல்களையும் பகிர்ந்து கொள்வோம்.</s>
    </P>
    <P id="10">
        <s id="10.1">இதற்கிடையில், எங்கள் டெவலப்பர் சமூகத்தின் தேவைகள் மற்றும் குரல்களின் அடிப்படையில் மேனிஃபெஸ்ட் வி 3 க்கு புதிய திறன்களை நாங்கள் தொடர்ந்து சேர்ப்போம். கடந்த சில மாதங்களில் கூட, விரிவாக்க தளத்திற்கு பல சுவாரஸ்யமான நீட்டிப்புகள் உள்ளன</s>
    </P>
    <P id="11">
        <s id="11.1">கூகிளின் கூற்றுப்படி, இது WebRequest ஐப் பயன்படுத்தும் செருகுநிரல்களில் தேவைப்படும் NetRequest அறிவிப்பு திறன்களைச் செயல்படுத்துவதில் தொடர்ந்து வேலை செய்கிறது, மேலும் புதிய API ஐ ஏற்கனவே உள்ள செருகுநிரல் டெவலப்பர்களின் தேவைகளை முழுமையாக பூர்த்தி செய்யும் வடிவத்தில் கொண்டுவர உத்தேசித்துள்ளது.</s>
    </P>
    <P id="12">
        <s id="12.1">வரவிருக்கும் மாதங்களில், மாறும் வகையில் உள்ளமைக்கக்கூடிய உள்ளடக்க ஸ்கிரிப்ட்களுக்கான ஆதரவையும், புதிய திறன்களுடன் நினைவக சேமிப்பு விருப்பத்தையும் வெளியிடுவோம். இந்த மாற்றங்கள் சமூக பின்னூட்டங்களை மனதில் கொண்டு வடிவமைக்கப்பட்டன, மேலும் டெவலப்பர்களால் அதிக தகவல்கள் பகிரப்படுவதால் நாங்கள் தொடர்ந்து அதிக சக்திவாய்ந்த நீட்டிப்பு ஏபிஐ செயல்பாட்டை உருவாக்குவோம்.</s>
    </P>
    <P id="13">
        <s id="13.1">எடுத்துக்காட்டாக, கூகுள் ஏற்கனவே சமூகத்தின் விருப்பங்களை கணக்கில் எடுத்துக்கொண்டு, பல நிலையான விதிமுறைகள், ரீஜெக்ஸ் வடிகட்டுதல், HTTP தலைப்புகளை மாற்றுதல், விதிகளை மாறும் மற்றும் மாற்றியமைத்தல், அளவுருவை நீக்குதல் மற்றும் மாற்றுதல். மற்றும் அமர்வு-குறிப்பிட்ட விதி தொகுப்பு உருவாக்கம்.</s>
    </P>
    <P id="14">
        <s id="14.1">வரவிருக்கும் மாதங்களில், உள்ளடக்க செயலாக்கத்திற்கான மாறும் தனிப்பயனாக்கக்கூடிய ஸ்கிரிப்டுகளுக்கான ஆதரவை மேலும் ரேமில் தரவைச் சேமிக்கும் திறனை மேலும் செயல்படுத்த திட்டமிடப்பட்டுள்ளது.</s>
    </P>
