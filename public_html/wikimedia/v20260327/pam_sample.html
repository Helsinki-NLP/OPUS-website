 <s id="22">2.8 Routers</s>
 <s id="23">Routers are used in optical networks for routing the traffic, be it a lightpath or packet traffic, from the source node to the destination node passing through the intermediate network nodes.</s>
 <s id="24">The routers used are of different types in both structure and technology.</s>
 <s id="25">In the following we give the types of routers commonly used in networks.</s>
 <s id="26">2.7.1 Mach‐Zehnder Interferometer (MZI) Multiplexer</s>
 <s id="27">2.8.1 Static Wavelength Router</s>
 <s id="28">In the wavelength switched routing networks the lightpaths have to be routed over the fiber between the source‐destination nodes.</s>
 <s id="29">Signals from different input ports of the router are sent to different output ports based on the wavelengths to route the lightpath to their destination.</s>
 <s id="30">The (N × N) wavelength router requires two blocks to build: first, N number of (1 × N) wavelength DMUX, and second, N number of (N × 1) wavelength multiplexer.</s>
 <s id="31">For a (N × N) static or non‐reconfigurable wavelength router, each of the N input fibers carries a set of {λ1,λ2 …λN} wavelengths and is connected to a (1 × N) DMUX.</s>
 <s id="32">The DMUX spatially separates the wavelengths and send them on N output ports.</s>
 <s id="33">The output stage has again N number (N × 1) MUXs, each of which is identical to DMUX but used in the opposite direction.</s>
 <s id="34">Each of the N number MUX combines the different λs connections coming from the N number DMUX and connects them to the output fiber.</s>
 <s id="35">A static wavelength router is also known as a static wavelength cross‐connect (WXC) in the wavelength‐routed networks.</s>
 <s id="36">In the static WXC, the cross‐connection pattern is fixed at the time when the device is made and cannot be changed dynamically.</s>
 <s id="37">Figure 2.18 illustrates a 4 × 4 wavelength static router as an example.</s>
 <s id="38">The connection between the two devices is such that different λs from different ports are combined by the MUX.</s>
 <s id="39">The N2 fibers between the input and output stages are connected in a way that prevents identical wavelengths from different input ports from being combined on the same output port, thus avoiding interference among these channels.</s>
 <s id="40">Hence, the path taken by any signal through the node is determined uniquely by its wavelength and port number.</s>
 <s id="41">The particular input port and its wavelength get routed to the particular output port depending on a routing matrix characterizing the router; this matrix is determined by the internal connections between the DMUX and the MUX.</s>
 <s id="42">An AWG can be made to work as a static wavelength router as PLC (Planar lightwave circuit) with silica on silicon technology.</s>
 <s id="43">2.8.2 Reconfigurable Wavelength Router</s>
 <s id="44">In order to have dynamic traffic connectivity in a wavelength routing network it is essential to have reconfigurability.</s>
 <s id="45">As discussed in the last sub‐section, the routing for a static router cannot be changed once it is designed.</s>
 <s id="46">We can get reconfigurability in the static wavelength router by adding a space switching stage between the DMUX and MUX columns to form the dynamic wavelength‐routing router.</s>
 <s id="47">This is also referred to as a wavelength selective cross‐connect (WSXC).</s>
 <s id="48">The switching stage can reconfigure the connectivity between the DMUX and MUX by electronically controlling the switching wavelength.</s>
 <s id="49">Figure 2.19 gives the functional block diagram of the N × N reconfigurable router [34].</s>
 <s id="50">Each incoming fiber connected to the DMUX carries N number WDM wavelengths.</s>
 <s id="51">The de‐multiplexed N wavelengths are directed to an array of N, N × N optical switch fabric between the DMUX and MUX.</s>
 <s id="52">Each (N × N) switch fabric is for a particular wavelength; hence all signals on a particular wavelength are directed to the same switch fabric.</s>
 <s id="53">The switched signals are then directed to the MUXs which are associated with the output ports.</s>
 <s id="54">The optical routing space switches in their simplest form may be made of (2 × 2) two state switches with no wavelength selectivity.</s>
 <s id="55">The 2 × 2 cross‐point switches may be reconfigured to adapt to changing traffic requirements by the control signaling.</s>
 <s id="56">Large space switch matrices/fabrics are formed by the interconnection of (2 × 2) or low port count space switches.</s>
 <s id="57">As the reconfigurable wavelength routers provide additional control in setting up connections, they are more flexible than static routers.</s>
 <s id="58">The routing is a function of both the wavelength chosen at the source node and the configuration of the switches in the network nodes.</s>
 <s id="59">2.8.3 Optical Packet Routing Switches</s>
 <s id="60">Lightpath routing, which was done in static and reconfigurable routers, is used for circuit switching in which the routing of traffic is done after the path has been established.</s>
 <s id="61">In the case of packet switching the packet itself carries the path ID.</s>
 <s id="62">The packets have header bits with source‐destination address, and each wavelength channel carries time division multiplexed information packets.</s>
 <s id="63">The packets, which are typically of random variable length, arrive at variable random rates from different sources and are not synchronized in time.</s>
 <s id="64">In electronic packet switching, the header bits are read in the electronic processor once there is an O‐E conversion at the node.</s>
 <s id="65">But in the case of optical packet switching, the packets contending for transmission on a given output line at a node may require optical buffering to resolve their contention.</s>
 <s id="66">Unlike the electronic storage, the available optical buffer/storage is limited.</s>
 <s id="67">It can be provided either with an optical delay line or by deflection routing or rerouting.</s>
 <s id="68">In most optical packet switches (OPSs), the header processing and switching functions are carried out electronically and the routing of the payload is done in the optical domain.</s>
 <s id="69">Several packet switches are proposed which have the data part fully optical but the control of switching operation is performed electronically [34–37].</s>

 <s id="71">The two input signals at two WDM wavelengths enter the 3 dB 2 × 2 coupler.</s>

 <s id="74">2.9.1.1 Permutation Switching Fabric</s>
 <s id="75">The interconnection pattern between the different input ports to the output ports of the switching fabric is represented by a connection matrix.</s>
 <s id="76">The interconnection between the switches in the case of N × N permutation switching fabric is such that there can be only point‐to‐point (one input port to another output port) unicast connection.</s>
 <s id="77">It cannot have broadcasting (1 × N) or multicasting (N × 1) connections.</s>
 <s id="78">The permutation fabric matrix, therefore, will have one cross‐point closing in each row and column.</s>
 <s id="79">A connection matrix for the 4 × 4 permutation fabric is shown in Figure 2.20a.</s>
 <s id="80">The switching matrix is square in this case as there can be only permutation connections possible.</s>
 <s id="81">The total possible input/output connection patterns or states are N! in a permutation switch fabric.</s>
 <s id="82">2.9.1.2 Generalized Switching Fabric</s>
 <s id="83">In the case of generalized switching fabric one can make all types of input/ output connections, i.e. point‐to‐point, point‐multipoint (1 × N), or multicasting (N × 1).</s>
 <s id="84">In general, it is an (N × R) matrix with 2NR connecting states.</s>
 <s id="85">Though the permutation switch has only one point closing in a row or column of the connection matrix, for generalized fabric switch there can be multiple points closing in a row or column depending on whether it has broadcasting or multicasting connections.</s>
 <s id="86">Unlike the permutation switch, which has only the control for bar and cross state, the generalized matrix has additional controls to specify the number of connections, as shown in Figure 2.20b for a 4 × 4 matrix.</s>
